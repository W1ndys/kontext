# 文档二：Kontext 初始代码设计方案（修订版 / 面向第一行代码）

## 1. 当前开发目标

当前不是开发完整 Kontext，而是开发它的第一个可用主链路：

> `kontext pack "<task>"`
> 读取 `.kontext/` + 收集候选上下文 + 调用 LLM + 生成高质量 Markdown Prompt 文档 + 保存到 `.kontext/prompts/`

同时提供两个辅助命令：

- `kontext init`
- `kontext validate`

---

## 2. 第一阶段功能范围

### 必做功能

1. `kontext init`

- 创建 `.kontext/`
- 创建默认 YAML 模板
- 创建 `.kontext/prompts/`

2. `kontext validate`

- 校验 `.kontext/` 是否存在
- 校验 YAML 是否可解析

3. `kontext pack "<task>"`

- 加载 `.kontext/`
- 收集候选上下文
- 构造 LLM 请求
- 调用 LLM
- 生成 Markdown Prompt
- 保存到文件

### 暂不做

- 剪贴板复制
- 自动更新物料
- 全仓智能语义检索
- 多 provider 路由
- 自动执行生成代码
- watch 模式

## 4. 模块职责设计

## `cmd/`

职责：

- 接收命令参数
- 初始化依赖
- 调用对应内部模块
- 打印结果和错误

包含：

- `root.go`
- `init.go`
- `pack.go`
- `validate.go`

约束：

- 不写打包逻辑
- 不写 LLM 调用细节
- 不直接拼接 Prompt 文本

---

## `internal/schema/`

职责：

- 定义 `.kontext/` YAML 结构
- 加载 `.kontext/` 内容
- 组合成 `Bundle`

核心能力：

```go
func LoadBundle(root string) (*Bundle, error)
```

---

## `internal/fileutil/`

职责：

- 文件读写
- 目录存在性检查
- 读取候选源码
- 构建基础目录摘要

---

## `internal/llm/`

职责：

- 封装 OpenAI-compatible API
- 接收标准化请求
- 返回标准化响应

目标：

- 将 LLM 调用隔离，便于未来替换 provider

---

## `internal/packer/`

职责：

- 实现 `pack` 主链路
- 匹配任务相关模块
- 收集候选上下文
- 生成 LLM 输入
- 请求 LLM
- 返回最终 Markdown 文本

它是当前版本最核心的模块。

---

## `internal/promptdoc/`

职责：

- 根据 task 生成 slug
- 生成文件名
- 保存 Markdown 文本到 `.kontext/prompts/`

---

## 5. 第一阶段关键数据结构

### Bundle

```go
type Bundle struct {
Manifest ProjectManifest
Architecture ArchitectureMap
Conventions Conventions
Contracts []ModuleContract
}
```

### ProjectManifest

```go
type ProjectManifest struct {

Purpose string `yaml:"purpose"`
Type string `yaml:"type"`
Language string `yaml:"language"`
GoVersion string `yaml:"go_version"`
Entrypoint string `yaml:"entrypoint"`
```
